## Slide 3 — System snapshot
### Title
**Our system in one glance**

### Lead line
We combined behavior, context, and ranking-aware modeling into one shift-aware decision pipeline.

### Blocks
**Observe**  
Radar trajectories + environmental context

**Describe**  
Kinematics + signatures + catch22 + validated external signal

**Rank**  
Ranking-aware ensemble + multiclass probability model

**Support**  
Probabilistic bird-group ranking for expert review

### Constraint callout
The hardest issue was not only imbalance.
It was seasonal / month shift.

### Margin notes
- radar is an ecological sensor
- migration behavior shapes the data
- decision support, not autopilot

## Slide 5 — What we learned
### Title
**Three lessons changed the project**

### Card 1
**Optimize the ranking**  
Macro-mAP rewards ordering, so our modeling had to match the metric.

### Card 2
**Treat shift as real behavior**  
Month shift reflected migration and operating variation, not just dataset noise.

### Card 3
**Keep ecology inside the model**  
Movement, radar, and context only helped when they stayed tied to mitigation meaning.

### Margin note
Generalization > local gain
