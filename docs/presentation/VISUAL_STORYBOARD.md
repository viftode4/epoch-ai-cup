# Visual Storyboard — AI Cup Congress Pitch

## Recommended art direction
**Primary recommendation:** Industrial Signal

Why:
- matches DeFabrique's industrial heritage vibe
- feels serious and contemporary without becoming generic tech-template purple
- supports ecology + energy + rigorous AI in one palette
- makes your workflow look like a strength, not admin

## Core palette
- Background: charcoal / graphite
- Accent 1: signal orange = urgency / energy / migration risk
- Accent 2: teal = ecology / validation / trust
- Support neutrals: warm off-white, slate gray

## Visual metaphors by slide
### 1. Opening
Use overlapping fields / orbs / radar-like geometry.
Message: biodiversity + wind energy + deployable AI intersect.

### 2. Problem
Three-card frame:
- Ecology
- Operations
- Monitoring challenge
This makes the problem feel multi-stakeholder immediately.

### 3. Workflow
Use a left-to-right research pipeline.
Visualise the team as builders with judgment.
This is where the "most submissions" award becomes a credibility signal.

### 4. Architecture
Use a four-stage system flow:
inputs -> features -> models -> decision support.
Add a side callout for seasonal shift as the real constraint.

### 5. Main insights
Use three large insight cards.
This is where you show that the workflow produced real conceptual learning.

### 6. Why trust it
Use four proof blocks:
- finalist status
- experiment scale
- submissions award
- honest evaluation
This slide sells the team as much as the result.

### 7. Deployment
Use an operational flow rather than a technical flow.
Radar monitoring -> AI ranking -> expert review -> targeted mitigation.
This is where the room sees beyond the competition.

## Animation ideas
- slow floating accent shapes on intro and close
- subtle reveal on each pipeline step
- progressive highlight of one insight at a time
- gentle pulse or glow on the seasonal-shift constraint box
- step-by-step operational flow reveal on final deployment slide

## What to avoid
- cluttered benchmark tables
- walls of bullets
- too many model names on screen
- visual style that looks like a default startup template
