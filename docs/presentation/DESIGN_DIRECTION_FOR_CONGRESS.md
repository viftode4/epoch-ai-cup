# Design Direction — Math Notebook Airspace

## New recommended refinement
**Math Notebook Airspace**

This is the hybrid of:
- clean data-forward structure
- stronger thematic figures
- more notebook / technical / sketched reasoning energy

## What changed
Compared with the previous hybrid, this version adds:
- math-grid / notebook-paper feel
- IBM Plex Mono labels and annotations
- hashed and sketched figure language
- more visible “systems thinking” inside the visual itself
- a stronger applied-research aesthetic

## Why this works for your case
It makes the team feel:
- nerdy in a good way
- technically serious
- research-driven
- visually specific
- still readable to a broad congress audience

## Typography
- **Headline:** Archivo
- **Body:** IBM Plex Sans
- **Annotations / system notes:** IBM Plex Mono

This triad is strong because it separates:
- big message
- readable explanation
- technical notebook detail

## Visual rules
- use one main figure per slide
- include notebook-style tags only where they help comprehension
- use hashed fills and light grid marks to suggest research reasoning, not clutter
- use orange for urgency / energy / migration pressure
- use teal for validation / trust / system control
- use warm neutrals to stay aligned with the fair-tech / civic vibe

## Best full-deck application
### Opening
Big message + airspace sketch + 2–3 tags max

### Workflow
A pipeline that feels like a lab notebook or systems diagram, not a business flowchart

### Architecture
Trajectories, side notes, and constraint tags

### Insight slide
Three cards plus one highlighted notebook-style annotation

### Proof slide
Evidence blocks with subtle hashed corners / lab-report framing

### Deployment slide
Operational flow with sketched turbine / monitoring framing
